## 个人效率

### 案例 1：智能晨间简报

**用户背景：** 互联网从业者，每天需要快速了解行业动态

**痛点：**
- 每天花 1 小时浏览多个网站和 App
- 信息过载，难以抓住重点
- 重要信息容易遗漏

**解决方案：**

```yaml
任务名称：晨间简报
执行时间：每天 08:00
触发方式：Cron 定时任务
输出渠道：飞书/微信
```

**配置步骤：**

1. **创建 Cron 任务**
```bash
openclaw cron add \
  --name "晨间简报" \
  --cron "0 8 * * *" \
  --tz "Asia/Shanghai" \
  --session isolated \
  --message "生成今日晨间简报" \
  --announce
```

2. **配置数据源**
在 SOUL.md 中添加：
```markdown
## 晨间简报内容
每天 8 点生成简报，包含：
1. 隔夜美股收盘情况（纳斯达克、标普 500）
2. 热门科技新闻（36 氪、虎嗅、TechCrunch）
3. GitHub Trending Top 5
4. 产品 Hacking News Top 3
5. 本地天气和空气质量
6. 日历上的今日重要会议

输出格式：
- 每条新闻一句话总结
- 附上原文链接
- 重要程度标记（⭐⭐⭐ 最重要）
```

3. **输出示例**
```
📊 晨间简报 - 2026-03-02

🌍 隔夜市场
- 纳斯达克 +1.2%，收于 18500 点 ⭐⭐
- 中概股普涨，纳斯达克金龙指数 +3.5% ⭐⭐⭐

📰 科技新闻
- OpenAI 发布 GPT-5 测试版，支持多模态理解 [链接] ⭐⭐⭐
- 小米汽车正式发布，售价 22.99 万起 [链接] ⭐⭐
- ...

💻 GitHub Trending
1. openclaw/openclaw - AI Agent 网关 ⭐⭐⭐
2. ...

🌤️ 北京天气
晴，15-25°C，空气质量良
适合户外运动

📅 今日安排
- 10:00 产品评审会
- 14:00 客户演示
- 16:00 团队 1:1
```

**效果：**
- 每天节省 45 分钟信息搜集时间
- 重要信息零遗漏
- 可以设置关键词，只关注特定领域

**用户反馈：**
> "用了两个月，已经离不开这个功能了。现在每天早上先看简报，再有重点地深入阅读。"

### 案例 2：自动记账系统

**用户背景：** 自由职业者，收入来源多，支出频繁

**痛点：**
- 忘记记账，月底对不上账
- 分类混乱，难以分析支出结构
- 发票管理困难

**解决方案：**

**系统架构：**
```
OpenClaw → ezBookkeeping API → 记账数据库
    ↓
飞书/微信（语音/文字/截图输入）
    ↓
自动分类 + 发票识别
```

**配置步骤：**

1. **部署 ezBookkeeping**
```bash
docker run -d -p 8080:8080 \
  -e EBK_SECURITY_ENABLE_API_TOKEN=true \
  -v $(pwd)/data:/ezbookkeeping/data \
  --name ezbookkeeping \
  mayswind/ezbookkeeping:latest
```

2. **安装记账技能**
```bash
git clone https://github.com/mayswind/ezbookkeeping-openclaw-skill.git
cd ezbookkeeping-openclaw-skill
openclaw skills install .
openclaw config set skills.ezbookkeeping.env.EBK_API_TOKEN "your_token"
openclaw config set skills.ezbookkeeping.env.EBK_BASE_URL "http://localhost:8080"
```

3. **自定义分类规则**
在 SOUL.md 中添加：
```markdown
## 记账规则
根据关键词自动分类：
- 餐饮：餐厅、外卖、咖啡、奶茶
- 交通：打车、地铁、公交、加油
- 购物：淘宝、京东、超市、衣服
- 娱乐：电影、游戏、KTV
- 工作：设备、软件、办公、差旅
- 居住：房租、水电、物业

收入分类：
- 项目收入：项目名称 + 收入
- 理财收益：基金、股票、利息
- 其他收入：红包、报销、退款
```

**使用方式：**

```
# 语音输入
"昨天在星巴克花了 35 块"
"滴滴打车 42 元"

# 文字输入
"记账：午餐 58 元，类别餐饮"
"收到客户 A 项目款 15000 元"

# 截图识别
[发送支付截图] → 自动识别金额、商家、时间
```

**月度报表：**
```bash
# 每月底自动生成
openclaw cron add \
  --name "月度财务总结" \
  --cron "0 9 1 * *" \
  --message "生成上月财务报告，包括收支对比、分类统计、趋势分析"
```

**效果：**
- 记账及时率从 30% 提升到 95%
- 每月节省 3-4 小时整理时间
- 清楚了解支出结构，优化消费习惯

```yaml
  2. 搜索相关项目和文档
  3. 生成会议要点清单
```

在 SOUL.md 中定义：
```markdown
## 会议提醒规则

2. 相关项目最新进展

4. 本次会议建议议程

时间：14:00-15:00
地点：3 号会议室
参会：张三、李四、王五

⏳ 待办跟踪
- [x] 完成需求文档
- [ ] 设计稿评审 ← 本次会议讨论
- [x] 技术方案确认

💡 建议议程
```

```markdown
## 会议纪要处理
会议结束后，发送会议录音或文字记录，自动：
1. 提取会议决议
2. 识别待办事项（负责人 + 截止时间）
3. 添加到任务管理系统
4. 发送会议纪要给参会人

示例指令：
"整理刚才的产品评审会纪要"

输出：

1. 确定 V2.0 版本上线时间为 3 月 15 日

下次会议：3 月 9 日 14:00
```

**效果：**
- 会议准备时间减少 50%

### 案例 4：自动站会机器人

**团队背景：** 远程办公的软件开发团队，10 人

**痛点：**
- 时区不同，难以找到共同的站会时间
- 站会流于形式，信息同步不充分
-  blockers 发现和解决不及时

**解决方案：**

**异步站会流程：**

```
每天 09:00（各时区本地时间）
  ↓
机器人私聊每个成员
  ↓
收集昨日进展、今日计划、障碍
  ↓
10:00 汇总发送到团队群
  ↓
团队成员 review，发现问题
  ↓
自动安排 1:1 讨论
```

**配置：**

```bash
# 站会提醒任务
openclaw cron add \
  --name "每日站会" \
  --cron "0 9 * * *" \
  --session isolated \
  --message "收集团队成员站会信息"

# 汇总任务
openclaw cron add \
  --name "站会汇总" \
  --cron "0 10 * * *" \
  --session isolated \
  --message "生成站会汇总报告"
```

**SOUL.md 配置：**
```markdown
## 站会规则
1. 每天 9 点私聊每个成员，询问：
   - 昨天完成了什么？
   - 今天计划做什么？
   - 遇到什么障碍？

2. 10 点汇总到团队群，格式：
📊 每日站会 - 2026-03-02

张三
✅ 昨天：完成登录模块开发
📅 今天：开始支付接口对接
🚫 障碍：支付文档不清晰，需要支持

李四
✅ 昨天：修复 3 个 bug
📅 今天：代码 review
🚫 障碍：无

...

⚠️ 需要关注
- 张三需要支付文档支持 → @王五 协助

3. 识别需要跟进的障碍，自动安排讨论
```

**效果：**
- 站会参与率从 70% 提升到 95%
- blockers 平均解决时间从 2 天降到 4 小时
- 团队成员更清楚彼此的工作进展

**解决方案：**

**数据自动采集：**

```markdown
## 周报数据源
1. GitHub：PR、Issue、Commit 统计
2. Jira/禅道：任务完成情况
3. 设计工具：设计稿数量和状态
4. 客服系统：用户反馈统计
5. 数据分析：产品指标变化
```

**自动生成流程：**

```bash
# 每周五 16:00 生成草稿
openclaw cron add \
  --name "周报生成" \
  --cron "0 16 * * 5" \
  --session isolated \
  --message "生成本周项目周报"

# 周一 09:00 提醒完善
openclaw cron add \
  --name "周报提醒" \
  --cron "0 9 * * 1" \
  --session main \
  --system-event "提醒完善周报"
```

**周报模板：**

```markdown
### 产品
- V2.1 需求评审完成
- 用户调研 5 场

### 开发
- 支付功能上线
- 性能优化，加载速度提升 30%

### 设计
- 新版首页设计稿完成

## 📅 下周计划
- V2.1 版本开发启动
- A/B 测试方案设计

## 🚫 风险和问题
- 服务器成本超预算 20% → 需要优化
- 设计资源紧张 → 需要协调

## 📝 其他
- 新入职 2 名工程师
- 参加产品大会分享
```

**效果：**
- 周报编写时间从 2 小时降到 20 分钟

### 案例 6：自媒体内容流水线

**用户背景：** 科技类自媒体作者，多平台运营

**痛点：**
- 选题困难，追热点慢
- 多平台分发耗时
- 数据分析繁琐

**解决方案：**

**内容创作流程：**

```
热点监控 → 选题评估 → 大纲生成 → 内容撰写 → 多平台分发 → 数据分析
   ↓           ↓           ↓           ↓           ↓           ↓
X/Twitter  评分系统   AI 生成    AI 初稿    自动发布   统一报表
Google 搜索  热度预测   结构优化   人工润色   格式调整   反馈分析
```

**热点监控配置：**

```bash
# 每小时监控一次
openclaw cron add \
  --name "热点监控" \
  --cron "0 * * * *" \
  --session isolated \
  --message "监控科技热点，评分并推荐 Top 3 选题"
```

**选题评估标准：**
```markdown
## 选题评分维度
1. 热度（40%）：搜索量、讨论度
2. 时效（30%）：新鲜度、持续性
3. 相关性（20%）：与账号定位匹配度
4. 差异化（10%）：能否写出新意

评分 > 80 分：强烈推荐
60-80 分：可以考虑
< 60 分：不建议写
```

**内容生成：**
```

1. 多模态支持
2. 技能市场上线
3. 移动端 Node 发布

## 实测体验
- 功能演示
- 优缺点分析

## 使用场景
- 个人效率提升
- 团队协作优化
- 企业应用案例

## 总结
- 适合人群
- 上手建议
```

**多平台分发：**
```markdown
## 平台适配规则
微信公众号：
- 标题：30 字内，吸引点击
- 格式：Markdown 转微信格式
- 配图：自动生成封面图

知乎：
- 标题：问题形式，如"如何评价..."
- 开头：添加"谢邀"
- 结尾：引导关注和点赞

Twitter/X:
- 拆分成 Thread
- 每条 280 字符内
```

**效果：**
- 选题时间从 2 小时降到 15 分钟

## 电商运营

### 案例 8：竞品价格监控

**用户背景：** 电商运营，需要监控竞品价格

**痛点：**
- 手动查看竞品价格耗时
- 价格变化响应慢
- 难以发现定价规律

**解决方案：**

**监控系统：**

```bash
# 每 2 小时检查一次
openclaw cron add \
  --name "价格监控" \
  --cron "0 */2 * * *" \
  --session isolated \
  --message "监控竞品价格，发现变化立即告警"
```

**监控配置：**
```markdown
## 监控规则
1. 监控平台：淘宝、京东、拼多多
2. 监控商品：竞品 Top 10
3. 监控频率：每 2 小时
4. 告警阈值：价格变化 > 5%

## 告警方式
- 小幅变化（5-10%）：飞书消息
- 大幅变化（> 10%）：电话 + 飞书
- 历史低价：立即通知
```

**输出报表：**
```
💰 竞品价格监控 - 2026-03-02 14:00

📊 价格变化
商品 A（竞品 X）
- 原价：199 元
- 现价：179 元（-10%）⚠️
- 历史低价：169 元

商品 B（竞品 Y）
- 原价：299 元
- 现价：299 元（无变化）

📈 价格趋势分析
- 竞品 X 每周末降价，平均降幅 8%
- 竞品 Y 价格稳定，偶尔参加平台活动

💡 建议
- 商品 A 建议跟进降价到 175 元
- 商品 B 保持原价，参加满减活动
```

**效果：**
- 价格响应时间从 1 天降到 2 小时
- 销售额提升 15%
- 定价策略更有针对性

```markdown
## 工单分类规则

### 问题类型
技术问题（40%）
- Bug 报告
- 功能咨询
- 集成问题

账单问题（25%）
- 发票申请
- 退款申请
- 套餐升级

产品建议（20%）
- 新功能需求
- 体验优化

其他（15%）
- 合作咨询
- 投诉建议

### 优先级判断
P0 - 紧急：系统宕机、数据丢失
P1 - 高：核心功能不可用
P2 - 中：非核心功能问题
P3 - 低：咨询、建议
```

**自动流转：**
```
用户提交工单
    ↓
AI 自动分类（类型 + 优先级）
    ↓
分配到对应团队
    ↓
P0/P1 立即通知负责人
P2/P3 进入工单池
    ↓
处理和跟进
    ↓
用户满意度调查
```

**效果：**
- 工单分类时间从 5 分钟降到 10 秒
- 分类准确率 92%
- 紧急问题响应时间从 4 小时降到 30 分钟
- 客户满意度提升 20%
